Choix techniques

Stack applicative

FastAPI + Python 3.12

Async I/O, validation Pydantic, OpenAPI auto (/docs), écosystème data/SQLAlchemy. Idéal pour API métier rapide à itérer.

React 18 + Vite + TS

Composants réutilisables, build rapide, typage strict. Build statique servi par nginx — image légère en K8s.

PostgreSQL 16

ACID pour stocks et mouvements ; relations catégories/produits ; même moteur en dev, CI et cluster.

Docker multi-stage

Reproductibilité : dev, CI et déploiement utilisent les mêmes Dockerfiles — pas de « ça marche chez moi ».

Choix techniques

Stack infrastructure

kind

Cluster K8s conforme CNCF sur la machine locale — démo Ingress, probes, ServiceMonitor sans coût cloud.

Helm 3

Paramétrage par values.yaml ; chart pfa-stock versionné avec l'app — template + lint en CI.

Terraform + Helm provider

Déploie charts et bucket S3 dans un seul plan ; reviewable en PR ; migrate vers AWS/EKS en changeant le provider.

LocalStack (S3)

API compatible AWS pour exports — démontre l'IaC cloud sans compte réel ni facturation.

kube-prometheus-stack

Standard de facto : Operator Prometheus, découverte via ServiceMonitor CRD — pas de config scrape manuelle.

Grafana + dashboard JSON

Dashboard « StockFlow API » en code (Git) ; injecté par Terraform ConfigMap — revue et reproductible.

CI/CD

Pipeline GitHub Actions — vue d'ensemble

6 jobs parallèles sur chaque push/PR main — feedback rapide, périmètres séparés (pas un seul job monolithique qui échoue tard).

helmlint · helm template

CI/CD — détail

Pourquoi chaque job existe

JobCe qu'il vérifieChoix & justification
backend ruff check · pytest -q Service container postgres:16 avec healthcheck — tests d'intégration réels (pas de mocks DB). Ruff = lint rapide, règles strictes avant merge.
frontend eslint · vitest --run Node 20 aligné sur l'image de build. Attrape régressions UI et erreurs de hooks React avant le build Docker.
docker docker build api + web Buildx sur runner Ubuntu propre — prouve que les Dockerfiles passent hors de ma machine. Web avec VITE_API_URL comme en prod.
terraform fmt -check · validate init -backend=false — pas de state distant en CI, mais syntaxe et providers validés à chaque PR.
helm helm lint · template Rendu des manifests K8s sans cluster — détecte YAML invalide, labels manquants, erreurs de values.
kubernetes kind-action + helm lint Cluster éphémère pfa-ci — vérifie que le chart est compatible API K8s réelle (pas seulement du lint statique).
