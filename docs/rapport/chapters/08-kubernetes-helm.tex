\section{Kubernetes avec minikube}

\subsection{Principe}

\textbf{Kubernetes} orchestre des conteneurs sur un cluster : scheduling, réseau, stockage, auto-healing via sondes. Pour la \textbf{validation du PFA}, le projet utilise \textbf{minikube} (driver Docker) : cluster local reproductible, addon ingress, déploiement Helm identique au job CI \texttt{kubernetes}.

\subsection{Démarrage}

\begin{lstlisting}[style=bash]
make prerequis-k8s
make cluster          # scripts/demarrer-cluster-minikube.sh
make demo-k8s         # cluster + smoke test in-cluster
\end{lstlisting}

Le guide \texttt{docs/demo-soutenance.md} détaille les entrées \texttt{/etc/hosts}, l'accès Grafana et \texttt{minikube dashboard}.

\subsection{Production optionnelle (VPS k3s)}

Un déploiement sur VPS (\texttt{docs/vps-setup.md}, \texttt{deploy.yml} manuel) reste documenté mais hors périmètre de la soutenance.

\section{Namespaces (minikube)}

Les scripts créent notamment :
\begin{itemize}
  \item \texttt{monitoring} — kube-prometheus-stack ;
  \item \texttt{pfa-stock} — application métier ;
  \item \texttt{kubernetes-dashboard} — visualisation du cluster ;
  \item \texttt{pgadmin} — pgAdmin4 (visualisation BDD PostgreSQL du cluster).
\end{itemize}

\section{Chart Helm \texttt{pfa-stock}}

\subsection{Values}

\begin{itemize}
  \item \texttt{values.yaml} — défauts (lint) ;
  \item \texttt{values-minikube.yaml} — minikube et CI : \texttt{imagePullPolicy: Never}, hôtes \texttt{pfa.test} / \texttt{api.pfa.test} ;
  \item \texttt{values-prod.yaml} — VPS optionnel (GHCR, sous-domaines publics).
\end{itemize}

\subsection{Ingress et routage (minikube)}

\begin{itemize}
  \item \texttt{pfa.test} → Service \texttt{web} ;
  \item \texttt{api.pfa.test} → Service \texttt{api}.
\end{itemize}

Le frontend minikube est construit avec \texttt{VITE\_API\_URL=http://api.pfa.test}.

\section{Observabilité et outils sur le cluster}

\begin{itemize}
  \item \textbf{kube-prometheus-stack} — Prometheus + Grafana (\texttt{grafana.test}) ;
  \item \textbf{Kubernetes Dashboard} — \texttt{minikube dashboard} ou port-forward ;
  \item \textbf{pgAdmin} — visualisation PostgreSQL du cluster (\texttt{pgadmin.test}) ; Adminer en Compose pour la démo Docker.
\end{itemize}

\section{Cycle de validation}

\begin{enumerate}
  \item \texttt{make cluster} — plateforme + application sur minikube ;
  \item CI — minikube éphémère, \texttt{helm upgrade}, smoke \texttt{/health} in-cluster ;
  \item Optionnel — \texttt{deploy.yml} vers VPS.
\end{enumerate}
