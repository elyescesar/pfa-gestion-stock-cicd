\label{ann:api}

Préfixe commun : \texttt{/api/v1}. Authentification : \texttt{Authorization: Bearer <jeton>} sauf connexion.

\section{Catégories}

CRUD standard : \texttt{GET/POST /categories}, \texttt{GET/PUT/DELETE /categories/\{id\}}. DELETE réservé admin.

\section{Produits}

\texttt{GET /produits}, \texttt{GET /produits/alertes}, \texttt{POST /produits}, \texttt{GET/PUT/DELETE /produits/\{id\}}.

Champs création : \texttt{nom}, \texttt{reference\_sku}, \texttt{quantite}, \texttt{seuil\_alerte}, \texttt{id\_categorie}.

\section{Mouvements}

\texttt{GET /mouvements}, \texttt{POST /mouvements} avec \texttt{type\_mouvement} (\texttt{entree}|\texttt{sortie}), \texttt{quantite}, \texttt{id\_produit}, \texttt{motif} optionnel.

Erreur métier : \texttt{400} \texttt{stock\_insuffisant}.

\section{Tableau de bord}

\texttt{GET /tableau-de-bord} — JSON avec compteurs et liste \texttt{produits\_alerte}.

\section{Santé et métriques}

\texttt{GET /health} — \texttt{\{"statut":"ok"\}} (ou équivalent).

\texttt{GET /metrics} — format Prometheus texte.

\section{Exemples de payloads JSON}

\subsection{Connexion}

Requête :
\begin{lstlisting}[language=Python]
POST /api/v1/auth/connexion
{"email": "admin@stock.tn", "mot_de_passe": "Admin123!"}
\end{lstlisting}

Réponse \texttt{200} :
\begin{lstlisting}[language=Python]
{"jeton_acces": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."}
\end{lstlisting}

\subsection{Création de produit}

\begin{lstlisting}[language=Python]

\section{Codes d'erreur HTTP courants}

\begin{table}[htbp]
  \centering
  \caption{Erreurs API typiques}
  \label{tab:api-errors}
  \small
  \begin{tabular}{@{}clp{7cm}@{}}
    \toprule
    \textbf{Code} & \textbf{detail} & \textbf{Contexte} \\
    \midrule
    401 & \texttt{jeton\_invalide} / non authentifié & Jeton absent, expiré ou signature incorrecte \\
    403 & accès refusé & Rôle insuffisant (ex.\ export admin) \\
    404 & \texttt{produit\_introuvable} & ID produit inexistant \\
    400 & \texttt{stock\_insuffisant} & Sortie dépassant le stock disponible \\
    422 & validation Pydantic & Corps JSON mal formé ou champs manquants \\
    \bottomrule
  \end{tabular}
\end{table}

\section{Exemples cURL (mode Compose)}

Avec l'API sur \texttt{localhost:8000} et un jeton dans la variable \texttt{\$TOKEN} :

\begin{lstlisting}[style=bash]
curl -s http://localhost:8000/health
curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:8000/api/v1/tableau-de-bord | jq
curl -s -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"type_mouvement":"entree","quantite":10,"id_produit":1}' \
  http://localhost:8000/api/v1/mouvements
\end{lstlisting}

En production : \texttt{https://api.pfa.elyes.dev/api/v1/...}.
