\chapter{Conclusion et perspectives}
\label{ch:conclusion}

\section{Bilan}

Le projet \stockflow{} démontre qu'une application métier de gestion de stock peut être livrée avec une chaîne DevOps complète : développement local Docker Compose, production sur k3s (VPS dédié), infrastructure as code, observabilité et CI/CD GitHub Actions.

\section{Objectifs atteints}

\begin{itemize}
  \item Fonctionnalités métier : catalogue, mouvements, alertes, tableau de bord ;
  \item Développement local via \texttt{make dev} ;
  \item Production sur VPS isolé (sans interférence Coolify) ;
  \item CD automatique (GHCR + Helm) ;
  \item Grafana, pgAdmin et dashboard Kubernetes en sous-domaines TLS.
\end{itemize}

\section{Limites connues}

\begin{itemize}
  \item Cluster mono-nœud ;
  \item State Terraform local ;
  \item Tests CI encore majoritairement SQLite ;
  \item Identifiants de démo à renforcer en production réelle.
\end{itemize}

\section{Perspectives}

\begin{enumerate}
  \item Backend Terraform distant (S3 + DynamoDB) ;
  \item Haute disponibilité multi-nœuds ;
  \item GitOps (Flux / Argo CD) ;
  \item Tests e2e post-déploiement ;
  \item Notifications sur alertes stock.
\end{enumerate}

\section{Mot de fin}

\stockflow{} illustre l'unification code applicatif et infrastructure dans un même dépôt, avec une séparation claire entre machine de développement et VPS de production dédié.
