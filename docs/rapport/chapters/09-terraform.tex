\chapter{Infrastructure as Code — Terraform}
\label{ch:terraform}

\section{Objectifs de Terraform dans StockFlow}

Terraform décrit l'infrastructure de façon \textbf{déclarative} et \textbf{idempotente}. Dans ce projet, il provisionne la \textbf{plateforme Kubernetes} sur le VPS PFA (bootstrap unique) :
\begin{enumerate}
  \item ingress-nginx et cert-manager (TLS Let's Encrypt) ;
  \item kube-prometheus-stack (Prometheus + Grafana) ;
  \item pgAdmin et Kubernetes Dashboard ;
  \item dashboard Grafana applicatif via ConfigMap.
\end{enumerate}

Le chart applicatif \texttt{pfa-stock} est déployé par GitHub Actions (\texttt{deploy.yml}), pas par Terraform.

\section{Providers}

\texttt{providers.tf} configure Kubernetes et Helm avec \texttt{config\_path} pointant vers le kubeconfig de l'opérateur (\texttt{\textasciitilde/.kube/config} ou \texttt{KUBECONFIG}).

\section{Variables}

\texttt{variables.tf} expose notamment :
\begin{itemize}
  \item \texttt{grafana\_host}, \texttt{pgadmin\_host}, \texttt{dashboard\_host} ;
  \item \texttt{acme\_email} pour Let's Encrypt ;
  \item \texttt{grafana\_admin\_password}, \texttt{pgadmin\_password} (sensibles) ;
  \item \texttt{postgres\_password} (aligné avec le secret GitHub \texttt{POSTGRES\_PASSWORD}).
\end{itemize}

\section{Ressources par fichier}

\subsection{\texttt{kubernetes.tf}}

Crée les namespaces : \texttt{ingress-nginx}, \texttt{cert-manager}, \texttt{monitoring}, \texttt{pfa-stock}, \texttt{pgadmin}, \texttt{kubernetes-dashboard}.

\subsection{\texttt{cert-manager.tf}}

Release Helm \texttt{cert-manager} et \texttt{ClusterIssuer} \texttt{letsencrypt-prod} (HTTP-01 via ingress nginx).

\subsection{\texttt{helm.tf}}

\subsection{\texttt{grafana\_dashboard.tf}}

ConfigMap du dashboard JSON \texttt{pfa-stock-api} pour le sidecar Grafana.

\section{Exécution opérationnelle}

\begin{lstlisting}[style=bash]
cd infrastructure/terraform
terraform init
terraform apply -var="grafana_admin_password=..." -var="pgadmin_password=..."
\end{lstlisting}

À exécuter depuis une machine dont le \texttt{KUBECONFIG} cible le \textbf{VPS PFA dédié} (voir \texttt{docs/vps-setup.md}). Le state reste local (\texttt{terraform.tfstate}, gitignoré).

\section{Validation en CI}

Le job \texttt{terraform} : \texttt{init -backend=false}, \texttt{fmt -check}, \texttt{validate}. Aucun \texttt{apply} sur les runners GitHub.

\section{Isolation Coolify}

Le VPS hébergeant Coolify (production client) n'est \textbf{jamais} ciblé par ce Terraform. Seul le VPS PFA reçoit k3s et ces releases.
