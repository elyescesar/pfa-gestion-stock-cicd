\chapter{Infrastructure as Code — Terraform}
\label{ch:terraform}

\section{Objectifs de Terraform dans StockFlow}

Terraform décrit l'infrastructure de façon \textbf{déclarative} et \textbf{idempotente}. Dans ce projet, le répertoire \texttt{infrastructure/terraform/} documente la \textbf{plateforme Kubernetes} (ingress, monitoring, outils) pour un déploiement VPS optionnel.

Pour la \textbf{validation du PFA}, le bootstrap local repose sur les scripts minikube (\texttt{scripts/demarrer-cluster-minikube.sh}) qui reproduisent la même logique Helm sans \texttt{terraform apply} sur la machine de démo.

Le chart applicatif \texttt{pfa-stock} est déployé par Helm (CI minikube ou \texttt{deploy.yml} manuel), pas par Terraform.

\section{Providers}

\texttt{providers.tf} configure Kubernetes et Helm avec \texttt{config\_path} pointant vers le kubeconfig de l'opérateur (\texttt{\textasciitilde/.kube/config} ou \texttt{KUBECONFIG}).

\section{Variables}

\texttt{variables.tf} expose notamment :
\begin{itemize}
  \item \texttt{grafana\_host}, \texttt{pgadmin\_host}, \texttt{dashboard\_host} ;
  \item \texttt{acme\_email} pour Let's Encrypt (VPS) ;
  \item \texttt{grafana\_admin\_password}, \texttt{pgadmin\_password} (sensibles) ;
  \item \texttt{postgres\_password} (aligné avec le secret GitHub \texttt{POSTGRES\_PASSWORD}).
\end{itemize}

\section{Ressources par fichier}

\subsection{\texttt{kubernetes.tf}}

Crée les namespaces : \texttt{ingress-nginx}, \texttt{cert-manager}, \texttt{monitoring}, \texttt{pfa-stock}, \texttt{pgadmin}, \texttt{kubernetes-dashboard}.

\subsection{\texttt{cert-manager.tf}}

Release Helm \texttt{cert-manager} et \texttt{ClusterIssuer} \texttt{letsencrypt-prod} (HTTP-01 via ingress nginx) — pertinent pour le VPS, pas pour minikube.

ConfigMap du dashboard JSON \texttt{pfa-stock-api} pour le sidecar Grafana. Le script minikube applique le même manifeste via \texttt{kubectl apply}.

\section{Exécution opérationnelle}

\subsection{Validation (CI et local)}

\begin{lstlisting}[style=bash]
cd infrastructure/terraform
terraform init -backend=false
terraform fmt -check
terraform validate
\end{lstlisting}

Le job \texttt{terraform} de la CI exécute ces commandes. Aucun \texttt{apply} sur les runners GitHub.

\subsection{Apply production (optionnel)}

\begin{lstlisting}[style=bash]
terraform init
terraform apply -var="grafana_admin_password=..." -var="pgadmin_password=..."
\end{lstlisting}

À exécuter depuis une machine dont le \texttt{KUBECONFIG} cible un VPS k3s (voir \texttt{docs/vps-setup.md}). Le state reste local (\texttt{terraform.tfstate}, gitignoré).
